HTTP = Hypertext transfer protocol, c'est un protocol de communication qui est prévu pour transmettre du texte,
il est donc utilisé pour le html.

Une requête HTTP débute toujours par la commande (GET,  POST, HEAD, PUT, DELETE) et contient des informations
sur la page demandée (version HTTP, type mine, navigateur etc) elle est aussi composée d'un header et parfois d'un corps.

Une réponse HTTP contient toujours un code d'état de la requête et fournit elle aussi des informations sur le serveur, 
la version du protocole HTTP et le type mine. Elle est constituée d'un header et d'un corps.

Les différentes requêtes possibles sont GET, POST, HEAD, PUT, DELETE.

GET → <chemin+requete>HTTP POST → <chemin>, la méthode GET envoie uniquement un header et la méthode POST envoie un header
et un corps.

On constate les échanges HTTP en php en récupérant les variables dans les tableaux $_GET et $_POST.

Oui, il est possible de modifier l'entete d'une requete via php.

L'utilisateur pourrait avoir des informations qui ne le concernent pas, pour éviter cela il faut sauvegarder 
l'identifiant de l'utilisateur tout au long de sa connexion et ne pas le transmettre via l'url → variables de session.

HTTPS = Hypertext transfer protocol secured avec du SSL. Il crypte l’envoi des requêtes entre le client et le serveur.

L'authentification est un procédé qui permet a un utilisateur d'accéder à du contenu personnalisé grâce a un nom 
d'utilisateur et un mot de passe. Pour cela, il faut mettre en place une interface de connexion.

L'utilisateur pourrait avoir accès a des informations confidentiels, afin d'éviter cela il faut utiliser le protocole 
https qui crypte les transmissions et empêche la récupération des identifiants.
